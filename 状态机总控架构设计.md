## Task 1: 创建设备适配器基础框架

### 1.1 `controller/devices/__init__.py`
- 定义 `DeviceBase` 抽象基类：`connect()`, `disconnect()`, `is_connected()`, `get_status()`
- 统一的设备状态枚举：`DISCONNECTED`, `CONNECTING`, `CONNECTED`, `ERROR`

### 1.2 `controller/devices/agv_adapter.py` — AGV 底盘适配器
封装 `agv_comm.AGVClient`，提供面向状态机的高层接口：
- `connect(ip, port)` / `disconnect()`
- `move_to(target_name)` — 前往目标点位，阻塞等待完成
- `move_to_floor(floor)` — 前往指定楼层（乘梯流程）
- `get_current_position()` — 获取当前楼层和位姿
- `get_battery()` — 获取电量
- `is_moving()` — 是否正在移动
- `emergency_stop()` — 急停
- `register_notification_handler(callback)` — 注册异常通知回调

### 1.3 `controller/devices/arm_adapter.py` — JAKA 机械臂适配器
封装 `JK_SDK.RC`，提供面向状态机的高层接口：
- `connect(ip)` / `disconnect()`
- `power_on()` / `enable()` — 上电使能
- `move_to_joint(joint_pos, speed)` — 关节运动到目标角度
- `move_to_pose(pose, speed)` — 笛卡尔运动到目标位姿
- `get_current_pose()` — 获取当前 TCP 位姿
- `get_joint_positions()` — 获取当前关节角度
- `set_digital_output(index, value)` — 控制 IO（可用于夹爪）
- `get_robot_state()` — 获取机器人状态
- `abort()` — 中止运动
- `go_home()` — 回到预设安全位

### 1.4 `controller/devices/vision_adapter.py` — 视觉相机（预留）
定义接口但返回模拟数据：
- `connect()` / `disconnect()`
- `capture()` — 触发拍照
- `detect_target()` — 检测放置目标，返回偏差坐标 `(dx, dy, dtheta)` 或 `None`

### 1.5 `controller/devices/gripper_adapter.py` — 夹爪（预留）
定义接口，底层可通过机械臂 IO 或独立串口控制：
- `connect()` / `disconnect()`
- `open()` / `close()`
- `is_gripping()` — 是否夹持中

## Task 2: 实现状态机核心

### 2.1 `controller/state_machine.py`

**状态定义（Enum）：**
```
IDLE                 — 在取餐点待命
DISPATCHING          — 从队列取出任务，准备出发
WAITING_STAFF        — 等待店员确认送出（已在取餐点）
MOVING_TO_ELEVATOR   — AGV 前往电梯口
TAKING_ELEVATOR      — 乘梯至目标楼层
NAVIGATING_TO_ROOM   — 导航至房间门口
VISION_CALIBRATING   — 视觉校准放置位置
PLACING_COFFEE       — 机械臂执行放置动作序列
RETURNING            — 返回取餐点（可能继续配送同楼层其他订单）
COMPLETED            — 单次任务完成
ERROR                — 异常状态，等待人工介入
CHARGING             — 低电量，前往充电点
```

**主循环逻辑：**
- 在后台线程运行，循环调用 `tick()` 方法
- `tick()` 根据当前状态执行对应动作并返回下一状态
- 状态转换时更新共享状态字典（供 Flask 读取）
- 异常捕获统一走 `on_error()` 处理

**关键状态转换：**
```
IDLE -> 队列非空且电量充足 -> DISPATCHING
DISPATCHING -> 取到任务 -> WAITING_STAFF（已自动等待）
WAITING_STAFF -> Flask dispatch 触发 -> MOVING_TO_ELEVATOR
MOVING_TO_ELEVATOR -> AGV 到达 -> TAKING_ELEVATOR
TAKING_ELEVATOR -> 到达目标楼层 -> NAVIGATING_TO_ROOM
NAVIGATING_TO_ROOM -> AGV 到达 -> VISION_CALIBRATING
VISION_CALIBRATING -> 视觉结果 -> PLACING_COFFEE
PLACING_COFFEE -> 放置完成 -> 队列中还有同楼层任务? NAVIGATING_TO_ROOM : RETURNING
RETURNING -> 到达取餐点 -> 队列非空? DISPATCHING : IDLE
```

### 2.2 `controller/task_manager.py` — 任务队列管理
- 从 Flask 的 `queue` 共享数据中取出任务
- 实现优先级排序（手动加急 > 超时提权 > 同楼层 > FIFO）
- 同楼层批量合并：将目标楼层相同的订单合并为一次行程
- 提供 `get_next_task()` / `peek_same_floor()` / `complete_task()` 接口

## Task 4: Flask 与状态机对接

修改 `server/app.py`：
- 导入并初始化状态机实例，在 Flask 启动前启动状态机后台线程
- `dispatch_order()` API：订单入队后通知状态机（而非直接操作 `lumi_status`）
- `get_lumi_status()` API：从状态机共享状态中读取真实数据（替代模拟数据）
- `get_queue()` API：从 `task_manager` 读取真实队列
- 新增 `GET /api/lumi/state` — 返回状态机当前状态（IDLE/DISPATCHING/...）
- 保留模拟告警生成（后续由状态机的异常回调产生真实告警）
